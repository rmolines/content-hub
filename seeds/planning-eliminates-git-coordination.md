## Raw context

From thesis.md — the insight that merge conflicts and branch strategy pain are actually planning failures. If file ownership is explicit in the plan, conflicts are prevented by design, not resolved after the fact. The planning phase assigns explicit file ownership per deliverable, making git coordination a non-issue.

## Why this matters

Challenges the assumption that git workflow complexity (branching strategies, merge tools, rebasing policies) is inherent to collaborative development. When agents execute in parallel, the planning phase must declare file boundaries — and that same discipline eliminates coordination overhead entirely. The article would argue that most "git problems" are symptoms of under-specified plans.
